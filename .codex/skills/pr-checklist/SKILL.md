---
name: pr-checklist
description: Run a lightweight PR review checklist when the user asks to review changes before merging.
---
PR checklist:
1) Summarize what changed in 3 bullets.
2) List files touched and why.
3) Risks: backward compatibility, config, secrets, security, performance.
4) If docs/README should change, propose exact edits.
5) Provide a merge recommendation: approve / request changes, with reasons.
